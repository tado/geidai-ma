## 本日の内容

本日の講義では、TouchDesignerを使用した動画ファイルの再生とミックスについて学びます。まず、演習で使用する動画素材として、Beepleが無償で公開しているFree VJ Loopsを紹介します。続いて、動画ファイル再生の基本操作として、映像の読み込み、再生、停止、ループといった基本的な扱い方を確認していきます。

その後、複数の映像をスイッチングする方法、キーボード入力によるインタラクションの導入、音に反応して映像を切り替える仕組み、そして操作をより直感的に行うための簡単なGUIの作成まで、段階的に内容を発展させていきます。

## TouchDesignerで使用可能な動画フォーマット

TouchDesignerの **Movie File In TOP** は、非常に幅広いコンテナ形式とコーデックに対応しています。制作の現場や用途に合わせて最適なものを選ぶことが重要です。

以下に、主要な対応フォーマットと推奨される用途をまとめました。


### 1. コンテナ形式（拡張子）
OS標準のメディア基盤（Windows: Media Foundation, macOS: AVFoundation）に加え、独自のデコーダーを搭載しているため、主要な拡張子は網羅されています。

* **.mov, .mp4, .m4v** (QuickTime / MPEG-4)
* **.avi, .mpg, .mpeg, .wmv** (旧来のWindows/汎用形式)
* **.mkv** (Matroska)
* **.dpx, .exr** (連番静止画的な中間形式)

### 2. 推奨される主要コーデック
TouchDesignerでのパフォーマンスを最大限に引き出すための代表的なコーデックです。

| コーデック | 特徴とメリット | 活用シーン |
| :--- | :--- | :--- |
| **Hap / Hap Q** | **GPUで直接デコード**するためCPU負荷が極めて低い。アルファチャンネル（透明度）保持が可能。 | VJ、大規模なプロジェクションマッピング、多画面出力。 |
| **H.264 / H.265** | 圧縮率が高くファイルサイズが小さい。**Hardware Decode** パラメータによりGPU支援を受けられる。 | Web素材、長尺の背景映像、ストレージ容量を抑えたい時。 |
| **Apple ProRes** | 高画質で編集耐性が高い。TouchDesigner 2023以降、Windows版でも公式にデコードが強化された。 | 放送用マスター、高品質なテクスチャ、ポストプロダクションとの連携。 |
| **NotchLC** | 高画質かつ低負荷。ProResとHapの中間のような位置付けで、10bit以上の高ビット深度にも対応。 | ハイエンドな映像演出、グラデーションの階調を重視する場合。 |
| **Cineform** | アルファチャンネルをサポートし、解像度の高い映像でも比較的軽量に動作する。 | アルファ付き素材の代替案として。 |

### 3. 静止画・その他のサポート
Movie File In TOPは、動画だけでなく以下の形式も読み込み可能です。

* **静止画:** .jpg, .png, .tif, .bmp, .dds, .exr, .hdr
* **連番画像:** フォルダ内の連番ファイルを動画のように再生可能。
* **Web URL:** http:// や https:// で始まるURLを直接指定して、ネットワーク上の画像や動画をロード可能。

### 4. 運用のポイント
* **パフォーマンスの確認:** パフォーマンスを重視する場合は、**Hap** または **NotchLC** が第一選択となります。
* **ハードウェアデコード:** H.264/H.265を使用する際は、パラメータの「Tune」ページにある **Hardware Decode** をオン（NvidiaやmacOS等の対応環境）にすることで、CPUの負荷を劇的に下げられます。
* **ビット深度:** 10bitや12bitの高ビット深度ファイルを扱う場合、Commonページの **Pixel Format** を Use Input にしておくと、情報が欠落せずに処理されます。

この動画では、TouchDesignerで各動画フォーマットを使用する際の負荷や、最適な書き出し設定について詳しく解説されています。
[TouchDesignerでの映像コーデックの選び方と設定](https://www.youtube.com/watch?v=hFhK6DGObYc)


## 動画素材の紹介

## 動画の再生の基本

### Movie File In TOP

TouchDesignerで動画を扱う際の中心となるのがMovie File In TOPです。このオペレーターでは、動画や静止画、連番画像、さらにはWeb上のリソースをURL指定で読み込み、TOPs（テクスチャ）として利用することができます。

対応フォーマットも非常に幅広く、.movや.mp4などの主要な動画形式はもちろん、Hap、NotchLC、Cineformといった高パフォーマンスなコーデックまでサポートされているため、リアルタイム再生に適した環境を柔軟に構築できます。再生モードも充実しており、タイムラインに同期させるLocked to Timeline、自由なタイミングで再生するSequential、任意のフレームを指定するSpecify Indexなど、用途に応じて切り替えることが可能です。また、Info CHOPを接続することで、解像度やFPS、デコードの状況、ドロップフレームの数といった詳細情報をリアルタイムに監視することもできます。



### Movie File In TOPで映像を再生

<img src="./img/movie-file-in-top.jpg" width="640">

[ダウンロード](https://github.com/tado/tdexamples/blob/main/03/01_playMovie.toe)

まずは実際に映像を再生してみましょう。Movie File In TOPに映像ソースを指定して読み込みます。このとき、ファイルパスは相対パスで設定しておくのがおすすめです。そうすることで、プロジェクト全体を別の場所に移動してもリンク切れを起こさずに済みます。

さらにTOP to CHOPを使ってInfo CHOPに接続すれば、現在の再生フレームや動画の長さといった詳細な情報を参照することができ、他のオペレーターと連動させる際の手がかりになります。パラメーターからは再生スピードや再生モードも自由に変更できるので、スロー再生や逆再生といった表現もここから簡単に試すことができます。



## 動画の再生の基本

ここからは、先ほど再生した動画をさらに発展させ、さまざまな方法で映像をスイッチングしたり、エフェクトをかけたりしていきましょう。具体的には、次の4つのステップで、基本から徐々に高度な表現へと進めていきます。最初はSwitch TOPを使って手動で映像を切り替えるところから始め、次にそれを自動化してランダムに切り替わるようにします。さらに、映像だけでなくエフェクトもランダムに適用されるよう発展させ、最後には音楽に反応して映像やエフェクトが切り替わるインタラクティブな仕組みへとつなげていきます。

1. Switch TOPによる映像の切り替え
1. Switch TOPのランダム自動切り替え
1. エフェクトもランダムに適用
1. 音楽 (音) に反応して切り替える



### Switch TOPによる映像の切り替え

<img src="./img/playMovie-switch.jpg" width="640">

[ダウンロード](https://github.com/tado/tdexamples/blob/main/03/02_playMovieSwitch.toe)

まずは、複数の映像を手動で切り替える仕組みを作成してみましょう。複数のMovie File In TOPを用意し、それらをすべてSwitch TOPに接続します。Switch TOPはindexの値によって、どの入力を出力として選択するかを決定する仕組みになっており、indexを0、1、2、3…と変化させることで接続された映像を順番に切り替えることができます。

さらに、単にindexを手で入力するだけでなく、ここで初めてインタラクションの要素を導入してみましょう。Keyboard In CHOPでキーボード入力を取得し、Count CHOPと組み合わせることで、キーを押すたびにカウントが増えて映像が切り替わるような仕組みを作ることができます。



### Switch TOPのランダム自動切り替え

<img src="./img/playMovie-autoswitch.jpg" width="640">

[ダウンロード](https://github.com/tado/tdexamples/blob/main/03/03_playMovieRandom.toe)

次に、手動での切り替えから一歩進めて、映像がランダムかつ自動で切り替わる仕組みを作ってみましょう。ここでは3つのCHOPを組み合わせます。Pattern CHOP（random）で乱数の列を生成し、LFO CHOP（Pulse）で一定の間隔のパルス信号を作り、Hold CHOPを使ってパルスが立ち上がった瞬間の乱数値を保持します。

こうして得られたランダムな整数値をSwitch TOPのIndexパラメーターにExportすれば、一定のリズムで映像がランダムに切り替わるシステムが完成します。LFO CHOPの周波数を変化させれば切り替わるテンポも自在にコントロールできます。



### エフェクトもランダムに適用

<img src="./img/playMovie-randAutoFx.jpg" width="640">

[ダウンロード](https://github.com/tado/tdexamples/blob/main/03/04_playMovieRandFX.toe)

映像のランダム切り替えに加えて、さらに適用するエフェクトもランダムに変化するようにしてみましょう。ここでは映像に対してエフェクト系のTOPを複数種類適用します。たとえばBloomで輝くような効果を加えたり、Edgeで輪郭抽出を行ったり、Transformで変形させたり、Levelでコントラストや透明度を調整したり、Lens Distortでレンズ歪みのような表現を加えたりといった具合です。

これらのエフェクトを並列に適用したうえで、Switch TOPを使ってそのうちの一つを選択する形にします。Switch TOPのIndexに対して、先ほどと同じようにランダムな値をExportすれば、映像の切り替えに合わせてエフェクトも次々に変化し、非常に表情豊かな映像表現を実現できます。



### 音楽 (音) に反応して切り替える

<img src="./img/playMovie-audioReact.jpg" width="640">

[ダウンロード](https://github.com/tado/tdexamples/blob/main/03/05_playMovieAudioReactive.toe)

最後に、時間による自動切り替えから一歩進んで、音楽に反応して映像やエフェクトが切り替わる、いわゆるオーディオリアクティブな仕組みに挑戦してみましょう。音の入力にはAudio File In CHOPを使って音声ファイルを読み込み、Audio Analysis COMPを通すことで、音の周波数帯域ごとのレベルを解析することができます。

特にキックドラム（低域）やスネア（中域）のように特徴的な音に反応させると、映像が音楽と同期しているかのような強い印象を生み出すことができます。解析された値をSwitch TOPのIndexやエフェクトのパラメーターに接続することで、音楽に合わせてダイナミックに変化するビジュアルを作り上げていきましょう。



## 本日の実習!

本日学んだ内容を踏まえて、Movie File In TOPを中心に動画を読み込み、自由な映像表現に挑戦してみましょう。素材としては、今回紹介したBeepleのFree VJ Loopsをそのまま活用するのはもちろん、自分自身で撮影・制作した動画素材や、著作権上問題のない手持ちの映像を使用しても構いません。

表現の方向性は自由です。複数の映像をミックスして新しい映像を作り上げたり、Switch TOPでダイナミックに切り替えたり、さまざまなエフェクトを重ねて独自の質感を探ったりと、これまでに学んだ要素を組み合わせてみてください。さらに余裕があれば、操作をより直感的に行えるようなユーザーインターフェイス（GUI）の開発にも挑戦してみましょう。完成した作品は次回の講義の冒頭で発表してもらいますので、自分なりのアイデアを形にしてみてください。
